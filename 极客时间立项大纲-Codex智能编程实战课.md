# Codex · 专栏大纲

**副标题**：动手篇——在真实工作流里把 Codex 用到过关  
**出品方**：袁从德  
**配套图书**：《Codex：AI 驱动的智能编程时代》（邮电出版社，可选延伸阅读）  
**配套代码**：https://github.com/congde/codexDemo.git  
**交付形态**：专栏  

---

## 一、课程概述

本专栏是 **Codex 动手篇**。不讲空泛概念，以**过关单元**组织内容——每个单元对应一次可验收的实操闭环（Brief → 委托 → 审 Diff → 跑验收），能复现才算学会。

- **实战载体**：[codexDemo](https://github.com/congde/codexDemo.git)（Lab、Challenge、验收脚本、`harness-kit`）
- **编排逻辑**：按**研发任务递进**——先出活、再设护栏、再打工单、再扩能力；**讲数随任务需要而定，不凑章节**
- **体量预期**：开篇词 + **约 17 讲** + 结束语（含 6 张核心工单）

### 专栏与图书的关系

同一作者、同一主题的两种交付形态，**互补而不重复**：

| | 图书《Codex：AI 驱动的智能编程时代》 | 本专栏（动手篇） |
|---|--------------------------------------|------------------|
| **回答什么** | 为什么、边界在哪、怎么判断 | 怎么配、怎么委托、怎么验收 |
| **组织方式** | 按知识模块系统展开（认知 → 工作流 → Harness → 场景 → 团队） | 按研发任务递进（先出活 → 设护栏 → 打工单 → 扩能力） |
| **学习形态** | 阅读 + 理解 + 举一反三 | 跟练 + 过关 + 提交验收产出 |
| **实战载体** | 以 `emotional_chat` 等项目为例讲解 | [codexDemo](https://github.com/congde/codexDemo.git) 工单 / Lab / 验收脚本 |
| **适合谁** | 想建立 Codex 工程化认知框架的读者 | 想把 Codex 用到日常研发工作流里的开发者 |

**怎么搭配**

- **只买专栏**：可以独立学完，按工单在 codexDemo 里练到过关。  
- **只读书**：能建立完整方法论，但动手深度依赖自己找项目练。  
- **书 + 专栏**：理解更稳、上手更快——书建立判断，专栏把判断落到命令与配置；二者内容不重叠，各学各的部分。

**一句话**：书是**地图**，专栏是**操练场**。

## 二、目标学员

想把 Codex **用到日常研发工作流里**的开发者与技术负责人——遇到问题知道开哪个入口、下什么委托、跑哪条验收，而不是只会聊天式改几行代码。

## 三、课程目录

开篇词｜先接到工单、再配工具——能复现才算学会

### 第一篇｜先出活——当天就要交差（3 讲）

> 过关标志：Fork codexDemo，`make check` 通过；独立完成 1 次委托 + 工单 #001

**讲 1**｜环境与第一次委托：codexDemo 导览、Brief → 任务 → 审 Diff → 验收命令  
**讲 2**｜App 工作台：线程、Plans、逐行审阅、分块 stage / revert  
**讲 3**｜工单 #001：热修复——完整闭环 + 验收脚本  

### 第二篇｜再设护栏——任务做大了再约束（2 讲）

> 过关标志：提交项目级 `config.toml`；`harness-kit` 联调通过

**讲 4**｜护栏装配 Lab：Sandbox / 审批 / AGENTS.md + Rules + Hook 一次装齐  
**讲 5**｜外部连接：Skills + MCP（GitHub / JIRA）+ 模型选型与账单  

### 第三篇｜工单歼灭——五张工单打穿工程链路（5 讲）

> 过关标志：完成 5 张工单中的至少 4 张

**讲 6**｜工单 #002：陌生模块勘察——地图 + 影响分析  
**讲 7**｜工单 #003：Bug 定位与回归——最小修改 + 测试  
**讲 8**｜工单 #004：跨端功能 PR——多文件改动 + 提交  
**讲 9**｜工单 #005：Review 双向闭环——findings 输出与改评  
**讲 10**｜工单 #006：CI 自修复——`codex exec` + GitHub Action  

### 第四篇｜并行与自动化——人不在线也能推进（2 讲）

> 过关标志：完成 1 次 Cloud Handoff；配置 1 条 Automation 并完成收件箱审批

**讲 11**｜执行环境与并行：Worktree / Cloud、Handoff、多线程、移动端跟进  
**讲 12**｜Automations 与 Slack 收件箱：定时唤醒、委派、审批、归档  

### 第五篇｜代码之外——Codex 不止写代码（2 讲）

> 过关标志：完成 1 次非编码任务并产出可验收文档

**讲 13**｜文档与调研：PDF / 表格 / 多源汇总 → Notion / Google Docs  
**讲 14**｜浏览器、GUI 与主动建议：标注、GUI 任务、跨工具待办  

### 第六篇｜团队与毕业——从个人熟练到可推广（3 讲）

> 过关标志：输出个人 Playbook（≤10 条委托模板 + 验收命令）

**讲 15**｜Worktree 三角色沙盘：Dev / Reviewer / On-call 分工  
**讲 16**｜企业治理与 SDK：`requirements.toml`、托管策略、接入自有工具链  
**讲 17**｜毕业实战：翻车复盘 + 综合 Challenge + Playbook 导出  

结束语｜过关的标准不是「听过」，而是「下次接到任务，知道开哪个入口、下什么委托、跑哪条验收」

## 四、单讲体例

每讲统一五环节：

| 环节 | 要求 |
|------|------|
| 实战 Brief | 背景、验收标准、禁止项（300–500 字） |
| 决策卡 | 何时用 / 何时不用 / 常见误用 |
| 实操演示 | App 或 CLI 完整走一遍，含配置片段与关键命令 |
| 翻车复盘 | 至少 1 个典型失败 + 恢复步骤 |
| 过关任务 | 1 个必做练习，产出截图或验收命令输出 |
