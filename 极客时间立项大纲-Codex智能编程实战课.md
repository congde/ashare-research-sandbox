# Codex 交付冲刺营 · 课程大纲

**副标题**：21 天工单驱动式 AI 编程实战——从第一条闭环到团队级护栏  
**出品方**：袁从德  
**配套图书**：《Codex：AI 驱动的智能编程时代》（邮电出版社，内容与本书约 70% 差异，互补联读）  
**配套代码**：https://github.com/congde/codexDemo.git  
**交付形态**：专栏（主）+ 嵌入录屏（辅）+ Challenge 卡片  
**总讲数**：36 讲 | 约 6–8 小时阅读量

---

## 一、课程概述

多数 AI 编程教程按「工具功能 → 参数说明 → 零散 Demo」展开，学员看完仍不知道接到工单该怎么下手。本书讲「为什么」与方法论全景；**本专栏不做书的朗读版**，而是模拟 Sprint：每天一张工单、一篇专栏、一段录屏、一次验收，练会「上下文 → 任务 → 验收」的交付节奏。

- **实战仓库**：codexDemo（全书唯一实战载体，含 Sprint 分支、预制缺陷与验收脚本）
- **核心方法**：Harness 工程、三步法验收、分级信任与审批模型  
- **工具锚点**：OpenAI Codex 为主，方法可迁移至 Cursor、通义灵码等

---

## 二、目标学员与学习收获

**目标学员**：工单型开发者、Tech Lead / EM、书读者、跨栈工程师。

**学习收获**：

1. 48 小时内完成 Sprint-1 首条闭环（安装 → 任务 → 验证 → 提交）
2. 装配护栏套件（项目说明 + Rules + Hook + 审批点），可 fork 到新仓库
3. 在五类工单场景中各完成一次闭环
4. 输出个人 Playbook（≤10 条可执行指令 + 验收命令）
5. 完成团队协作沙盘：角色分工、handoff、试点指标与退出条件

## 五、单讲体例

| 环节 | 体量 |
|------|------|
| Challenge Brief | 300–500 字：背景、验收标准、禁止项 |
| 正文 | 1500–2500 字：方法、决策卡、关键命令 |
| 嵌入录屏 | 5–10 分钟：codexDemo 分支实操 |
| Debrief | 400–600 字：3 条原则 + 1 条反模式 |
| 打卡 | Challenge ID + 验收命令截图 |
